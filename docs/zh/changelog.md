## [1.3.1] - 2025-01-27

### 新增
- **51 条新安全规则**：扩展了安全覆盖范围，新增了 5 个类别的 51 条规则：
  - 依赖安全（10 条规则）：过时包检测、已知漏洞、无人维护的包、大型包警告、已弃用的包、不安全的协议、git 依赖、本地依赖、缺失的对等依赖、生产环境中的开发依赖
  - 环境变量安全（5 条规则）：环境文件暴露、环境变量中的敏感数据、客户端暴露、敏感变量的默认值、环境变量日志记录
  - 认证与授权安全（10 条规则）：弱密码检测、不安全的哈希算法、缺失的认证检查、硬编码凭证、会话固定漏洞、弱令牌生成、缺失的注销功能、暴力破解漏洞、权限绕过风险、JWT 安全问题、OAuth 安全问题
  - 文件上传安全（10 条规则）：文件上传无验证、路径遍历漏洞、可执行文件上传、恶意文件检测、webshell 检测、大小限制违规、不安全的存储位置、文件名验证问题、扩展名验证问题、MIME 类型验证缺口、病毒扫描要求
  - API 安全（16 条规则）：不安全的 HTTP 使用、API 密钥暴露、缺失的认证、不安全的 SSL/TLS 配置、速率限制问题、输入验证缺口、错误处理问题、CORS 配置错误、API 版本控制问题、分页漏洞、GraphQL 注入风险、webhook 安全问题、缓存控制问题、幂等性问题

- **测试示例**：为新安全规则添加了 5 个新测试文件：
  - `new-rules-dependency-security.json`：依赖安全测试示例
  - `new-rules-env-security.js`：环境变量安全测试示例
  - `new-rules-auth-security.js`：认证和授权安全测试示例
  - `new-rules-file-upload-security.js`：文件上传安全测试示例
  - `new-rules-api-security.js`：API 安全测试示例

### 变更
- **文档更新**：
  - 在 README 中将安全规则数量从 149+ 更新为 220+
  - 将核心安全规则从 129 条更新为 200 条
  - 为新的安全规则类别添加了详细文档
  - 将测试文件数量从 36 个更新为 41 个

### 移除
- **临时文件**：移除了临时测试脚本和目录：
  - `test-new-rules.js`
  - `test-new-rules-functionality.js`
  - `tests/quick-test-new-rules.js`

### 修复
- **插件集成**：验证了所有插件（Vite、Webpack、Nuxt、VSCode）都已正确集成新的安全规则

## [1.3.0] - 2025-01-20

### 新增
- **高级语义分析**：基于 AST 的分析，提高了漏洞检测的准确性
- **依赖扫描**：集成了 npm audit 和自定义漏洞数据库
- **高级报告**：多格式报告（JSON、HTML、Markdown、SARIF），带有趋势分析
- **合规性检查**：支持 OWASP、GDPR、HIPAA、PCI-DSS、SOX 合规标准
- **内存优化**：大型项目的批处理和缓存
- **MCP 集成**：用于 AI 助手的模型上下文协议集成

### 变更
- **性能**：通过优化规则引擎，将扫描速度提高了 40%
- **规则引擎**：重构以提高可扩展性和自定义规则支持

## 版本格式

- **主版本**：破坏性变更或重大新功能
- **次版本**：新功能或重大改进
- **补丁版本**：错误修复或小改进

## 向后兼容性

我们努力在次版本之间保持向后兼容性。破坏性变更只会在主版本发布中引入，并将在更新日志中明确记录。
