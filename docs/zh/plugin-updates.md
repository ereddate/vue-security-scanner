# 插件更新指南

本指南提供了关于 Vue Security Scanner 插件更新的信息。

## 概述

插件更新对于以下方面至关重要：
- 访问新的安全规则和功能
- 保持与 Vue.js 版本的兼容性
- 修复错误和性能问题
- 确保符合安全标准

## 插件系统

Vue Security Scanner 支持多种类型的插件：

- **核心插件**：提供基本功能的内置插件
- **社区插件**：由社区贡献的第三方插件
- **企业插件**：企业许可证可用的高级插件

## 更新流程

### 核心插件

当您更新扫描器本身时，核心插件会自动更新：

```bash
# 更新 Vue Security Scanner
npm update -g vue-security-scanner

# 或对于项目特定的安装
npm update vue-security-scanner --save-dev
```

### 社区插件

社区插件必须单独更新：

```bash
# 更新特定的社区插件
npm update vue-security-plugin-example --save-dev

# 更新所有插件
npm update --save-dev
```

### 企业插件

企业插件通过企业门户进行管理：

1. 登录企业门户
2. 导航到插件部分
3. 检查可用更新
4. 下载并安装更新的插件

## 更新通知

### CLI 通知

扫描器会在扫描期间通知您可用的插件更新：

```
🔄 Plugin Update Available: vue-security-plugin-example v2.1.0 → v2.2.0
📦 Run: npm update vue-security-plugin-example to update
```

### 仪表板通知

Vue 安全仪表板提供更新通知：

1. 打开仪表板
2. 导航到插件部分
3. 查看可用更新
4. 直接从仪表板更新插件

## 插件更新最佳实践

### 测试更新

在生产环境中更新插件之前：

1. **在开发环境中测试**：在开发环境中使用更新的插件运行扫描
2. **查看更新日志**：检查更新中包含的更改
3. **运行回归测试**：确保现有功能仍然正常工作
4. **检查兼容性**：验证与您的 Vue.js 版本的兼容性

### 更新策略

- **定期更新**：安排定期插件更新（建议每月）
- **分阶段推出**：首先向一部分项目推出更新
- **备份配置**：在更新之前始终备份插件配置
- **版本固定**：考虑在 package.json 中固定插件版本以获得稳定性

## 插件更新故障排除

### 更新失败

如果插件更新失败：

1. **检查网络连接**：确保您有互联网访问
2. **验证 npm 注册表**：检查 npm 注册表是否可访问
3. **清除 npm 缓存**：运行 `npm cache clean --force`
4. **检查权限**：确保您对 node_modules 有写入权限

### 兼容性问题

如果更新后遇到兼容性问题：

1. **回滚到先前版本**：恢复到最后一个工作版本
2. **检查插件文档**：在文档中查找破坏性更改
3. **更新扫描器**：确保您使用的是最新的扫描器版本
4. **联系插件作者**：向插件维护者报告问题

### 性能问题

如果更新后性能下降：

1. **监控资源使用**：检查 CPU 和内存使用情况
2. **审查插件设置**：调整插件配置以提高性能
3. **禁用有问题的插件**：临时禁用插件以识别问题插件
4. **报告性能问题**：向插件维护者提供反馈

## 插件开发更新

如果您正在开发插件：

1. **遵循语义化版本控制**：为您的插件使用语义化版本控制

3. **更新文档**：保持插件文档最新
